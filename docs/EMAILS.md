# Eventiko — Emails (Resend) & deliverability

Eventiko sends three kinds of email, all through **Resend** (or SMTP):

1. **Confirm account** & **Reset password** (and magic‑link / invite / email‑change) — auth emails generated by Supabase, delivered via our branded **Send Email hook**.
2. **Tickets** — a per‑order, event‑specific confirmation with reference numbers + a download link, sent by the backend when an order is finalized.

---

## 1. Create a Resend account & verify your domain (critical for inbox placement)

1. Sign up at [resend.com](https://resend.com) and **add your domain** (e.g. `eventiko.com`).
2. Resend shows DNS records — add them at your DNS provider:
   - **SPF** (a `TXT` record authorizing Resend to send for your domain)
   - **DKIM** (CNAME/TXT records — cryptographically signs your mail)
   - **DMARC** — add a `TXT` record at `_dmarc.yourdomain.com`, e.g.
     `v=DMARC1; p=none; rua=mailto:dmarc@yourdomain.com` (move to `p=quarantine` once aligned).
3. Wait for Resend to show the domain as **Verified**.
4. Create an **API key** → set `RESEND_API_KEY` on Render.
5. Set `EMAIL_FROM="Eventiko <tickets@yourdomain.com>"`, `EMAIL_REPLY_TO=support@yourdomain.com`, `SUPPORT_EMAIL=support@yourdomain.com`.

> **Why this matters:** mail from an unverified domain (or a generic `@gmail.com` from address) is what lands in spam. SPF + DKIM + DMARC alignment on your own verified domain, a real reply‑to, and the plain‑text part we attach to every message are the things that keep Eventiko email in the inbox. The backend always sends a text alternative + reply‑to automatically.

---

## 2. Branded auth emails (confirm account / reset password) via the Send Email hook

This routes Supabase's auth emails through Resend with Eventiko branding.

1. In Supabase: **Authentication → Hooks → Send Email Hook** → **Enable**, set the URL to:
   `https://<your-backend>.onrender.com/api/hooks/email`
2. Copy the generated **secret** (looks like `v1,whsec_…`) → set it on Render as `SEND_EMAIL_HOOK_SECRET`.
3. Done. Supabase now calls our hook for sign‑up confirmation, password recovery, magic links, invites and email changes; the hook builds the verification link and sends a branded email via Resend. The link points back to `…/auth/v1/verify?…&redirect_to=<your-site>/auth/callback`, which our `/auth/callback` route exchanges for a session.

**Fallback (no hook):** if you prefer not to use the hook, set Supabase **Project Settings → Auth → SMTP** to Resend's SMTP credentials and customize the templates under **Authentication → Email Templates**. Either way, use your verified domain.

---

## 3. Ticket emails

Sent automatically by the backend when an order is finalized (`services/ticketing.ts` → `sendConfirmationEmail`). They are **event‑specific** (event title, order number, every ticket's reference) and link to the hosted tickets + PDF. No configuration beyond `RESEND_API_KEY` + `EMAIL_FROM`.

---

## 4. Test
- Sign up with a real address → you should receive the **Confirm account** email from your domain.
- Use **Forgot password** → **Reset password** email arrives.
- Buy a ticket (Stripe test card `4242 4242 4242 4242`) → **ticket** email arrives with references + download link.
- Check the message lands in the **inbox** (not spam) and that "signed‑by"/"mailed‑by" show your domain. Tools like [mail-tester.com](https://www.mail-tester.com) give a deliverability score.
